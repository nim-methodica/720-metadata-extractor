# זיהוי סוגי שאלות ומבנה answers/correctAnswers

לפי תקן 720, `questionType` הוא ערך יחיד מתוך: `fill-in` / `true-false` / `choice` /
`numeric` / `sequencing` / `matching` / `other`. המבנה של `answers` ו-`correctAnswers`
משתנה לפי הסוג. מסמך זה מסביר איך לזהות כל סוג מהתסריט ואיך לכתוב אותו נכון.

## עקרון בסיסי

לפני שמזהים סוג, הסתכל בשקף:

1. **מה רואה הלומד?** שדה הקלדה חופשי? רשימת אפשרויות? גרירה?
2. **מה המקום הפיזי של התשובה במסך?** האם יש תבנית מולידה (`__ : 1`)?
3. **מה נדרש להזין?** מספר טהור? טקסט? רצף? זוגות?

## choice — שאלת בחירה מרובת אפשרויות

**זיהוי**: הלומד רואה 3-5 אפשרויות טקסטואליות ובוחר אחת (או יותר).

**מבנה**:
```json
{
  "questionType": "choice",
  "questionText": "נוסח השאלה",
  "answers": ["אפשרות א", "אפשרות ב", "אפשרות ג", "אפשרות ד"],
  "correctAnswers": ["אפשרות ב"]
}
```

- `answers` — כל האפשרויות המוצגות.
- `correctAnswers` — מערך של תשובות נכונות. יכול להיות יותר מאחת (בחירה מרובה).
- **הטקסט של התשובה בדיוק כפי שמופיע במסך** (כולל אות סעיף "א." "ב." אם יש).

## true-false — שאלת נכון/לא נכון

**זיהוי**: שתי אפשרויות בלבד — "נכון" ו"לא נכון" (או "כן" ו"לא").

**מבנה**:
```json
{
  "questionType": "true-false",
  "questionText": "טענה שיש לקבוע אם היא נכונה",
  "answers": ["נכון", "לא נכון"],
  "correctAnswers": ["נכון"]
}
```

## numeric — תשובה מספרית טהורה

**זיהוי**:
- הלומד רואה שדה הקלדה יחיד.
- התשובה היא **מספר בלבד** (אין ":1", אין ק"מ, אין יחידה).
- לרוב יש תבנית במסך שמראה את היחידה בקבוע (למשל שדה + "ס"מ", או שדה + ": 1").

**מבנה**:
```json
{
  "questionType": "numeric",
  "questionText": "נוסח השאלה",
  "answers": [],
  "correctAnswers": [25]
}
```

- `answers` תמיד `[]`.
- `correctAnswers` מכיל **מספר** (לא מחרוזת). כתוב `25000` ולא `"25,000"`.
- הפלטפורמה תזהה אוטומטית `25000` וגם `25,000` כי היא משווה ערך מספרי.

**דוגמאות**:
- "מהי המסה בגרם?" → `25` (מספר)
- "חשבו את קנה המידה: __ : 1" → `25000` (המספר בלבד; ה-":1" קבוע במסך)
- "כמה מטרים?" → `4` (מספר)

## fill-in — תשובה מחרוזתית

**זיהוי**:
- הלומד מקליד טקסט חופשי (או מחרוזת מורכבת).
- התשובה **אינה מספר טהור** — כוללת מילים, יחידות, סמלים כמו ":1".
- אין תבנית קבועה במסך שמפרידה את המספר מהטקסט.

**מבנה**:
```json
{
  "questionType": "fill-in",
  "questionText": "נוסח השאלה",
  "answers": [],
  "correctAnswers": [
    "25,000 : 1",
    "25,000:1",
    "25000 : 1",
    "25000:1"
  ]
}
```

- `answers` תמיד `[]`.
- `correctAnswers` מכיל **מחרוזות**. אם תשובה יכולה להיכתב בואריאנטים שונים (עם/בלי פסיק,
  עם/בלי רווח סביב הנקודתיים) — הוסף כמה ואריאנטים.

## sequencing — סידור פריטים לפי סדר

**זיהוי**: הלומד גורר פריטים כדי לסדר אותם בסדר נכון. יש 3-6 פריטים עם מספרים 1, 2, 3...

**מבנה**:
```json
{
  "questionType": "sequencing",
  "questionText": "סדרו את שלבי X לפי הסדר",
  "answers": [
    "שלב א",
    "שלב ב",
    "שלב ג",
    "שלב ד"
  ],
  "correctAnswers": [
    "שלב א",
    "שלב ב",
    "שלב ג",
    "שלב ד"
  ]
}
```

- `answers` — כל הפריטים שנתונים ללומד (כולל כאלו שלא צריכים להיכלל בפתרון, אם הפריט
  דורש סינון).
- `correctAnswers` — הפריטים בסדר הנכון (בלי הפריטים שאמורים להישלף).

## matching — התאמה בין שני קבוצות

**זיהוי**: הלומד גורר פריטים מקבוצה אחת לקבוצה אחרת. בדרך כלל יש "מחסן" עם ערכים ושדות
ריקים לקבל אותם. **כולל גם**: שאלות עם 3+ שדות הקלדה מספריים (למשל "מדדו 3 פעמים",
"רשמו כל תוצאה") — נטפל בהן כ-matching, לא כ-numeric נפרד לכל שדה. **כולל גם**: שאלת
**תפריט נפתח (dropdown)** לבחירת ערך מתאים מרשימה — גם זו matching, גם אם ויזואלית זה
נראה כמו שדה בחירה יחיד (ולא כ-`choice`).

**מבנה**:
```json
{
  "questionType": "matching",
  "questionText": "גררו את X ל-Y",
  "answers": {
    "source": ["פריט א מקבוצת מקור", "פריט ב", "פריט ג"],
    "target": ["ערך 1", "ערך 2", "ערך 3", "ערך 4"]
  },
  "correctAnswers": [
    {"source": "פריט א מקבוצת מקור", "target": "ערך 2"},
    {"source": "פריט ב", "target": "ערך 4"},
    {"source": "פריט ג", "target": "ערך 1"}
  ]
}
```

- `source` — הפריטים שצריך למצוא להם התאמה.
- `target` — האפשרויות שאפשר לגרור אליהן.
- `correctAnswers` — מערך של אובייקטים עם זוגות `{source, target}`.

**אין חובה שכל source יתאים ל-target (אפשר להשאיר בלי התאמה) — לפי התקן**.

**⚠️ מקורות שהם תמונות/גרפיקה (למשל: "גררו תיאור מתאים ל-4 מטרות קליעה", גרפים, סדרות
מדידה מוצגות כתמונה) — עצור ושאל את המשתמש, אל תנחש.**

`source` חייב להכיל את **המזהה הנקי** בלבד, כפי שהוא מופיע כטקסט בשקף (למשל: `"מטרה 1"`,
`"מטרה 2"`, `"סדרה 1"`) — **לעולם אל תשתול לתוכו פרשנות ויזואלית מנוחשת** (לא `"מטרה 1 -
חיצים מפוזרים סביב המרכז"`). הפרשנות של מה מוצג בכל תמונה שייכת אך ורק ל-`correctAnswers`,
ורק אם אפשר לקבוע אותה בוודאות מהטקסט שחולץ.

אם **התוכן שקובע את הזיווג הנכון (`correctAnswers`) מופיע רק בתמונה/גרף/דיאגרמה ואין לו
תיאור טקסטואלי מקביל** ב-`slides.txt` — זה בדיוק המקרה היחיד שבו אסור להמשיך לבד:
1. **עצור** את העיבוד של הפריט הזה.
2. **תאר למשתמש** בדיוק מה חסר (למשל: "בפריט X יש שאלת matching בין 4 'מטרות קליעה' —
   התמונות לא מכילות תיאור טקסטואלי של הדפוס בכל מטרה. מה מוצג בכל אחת, ומה הזיווג
   הנכון ל-4 היעדים?").
3. **המתן לתשובת המשתמש** ומלא את `correctAnswers` לפיה.

**אסור**:
- לנחש פרשנות ויזואלית סבירה-אך-לא-מאומתת ולהציג אותה כעובדה.
- להשאיר `correctAnswers: []` ולהמשיך הלאה בלי לשאול — הפלט הסופי חייב להיות מלא.

זהו יוצא הדופן היחיד לעיקרון "אפס שאלות למשתמש" של הסקיל — ראה `conventions.md`.

## other — כל השאר

אם השאלה לא נופלת באחת מהקטגוריות למעלה (למשל: ציור, הקלטה, בחירת כפתור באנימציה),
השתמש ב-`other`. נסה קודם לזהות אם באמת אין התאמה — הרוב הגדול נופל תחת הקטגוריות
הרגילות.

## שאלות "ללא מענה מוערך" (Selected במקום Answered)

חלק מהפריטים אינם דורשים תשובה מוערכת אלא רק **בחירה** של הלומד (בחירת דמות, בחירה בין
פלייליסטים, בחירת מצב לימוד). לפי תקן xAPI זו פעולה מסוג `Selected` ולא `Answered`.

**איך לרשום במטא-דאטה של הפריט:**
- אם אין שאלה כלל (רק בחירה טכנית): `"questions": []`.
- אם יש בחירה שהיא חלק מרצף מוערך (חימום כלשהו): רשום אותה כשאלה רגילה עם `choice`.

## שאלות עם סעיפים מרובים

פריט אחד יכול להכיל 2, 3, או 4 סעיפים (א', ב', ג', ד') — כל אחד כשאלה נפרדת ב-
`questions[]`. סמן את מזהי השאלות `q1, q2, q3, q4` ואת נוסח השאלה עם הסעיף:

```json
"questions": [
  {
    "questionId": "q1",
    "questionText": "א. חשבו X",
    ...
  },
  {
    "questionId": "q2",
    "questionText": "ב. חשבו Y",
    ...
  }
]
```

זה נכון גם אם כל השאלות באותו סוג וגם אם הן משלבות סוגים שונים.
